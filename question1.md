1. Why does regular committing support software quality?
becasue regular committing supports software quality because it creates small, trackable changes that are easier to review, debug, and roll back if something breaks. It also encourages developers to test and validate their work frequently rather than letting untested code pile up.

2. How can Git history provide evidence of individual progress?
Git history provides evidence of individual progress by showing a timestamped log of each developer's commits, including what was changed, when, and by whom, making it easy to track contributions, work patterns, and how a project evolved over time.
